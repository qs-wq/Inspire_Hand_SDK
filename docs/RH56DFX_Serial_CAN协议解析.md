## 1. 硬件接口

### 1.1 接口类型
- **连接器**：GX12 航空插头端子
- **通信协议**：CAN / CANFD

### 1.2 默认波特率

| 模式 | 仲裁段波特率 | 数据段波特率 |
|------|-------------|-------------|
| CAN | 1000K | - |
| CANFD | 1M | 5M |

### 1.3 PC 端通信方式（Serial-CAN）

灵巧手本体使用 CAN 总线，PC 侧通常通过 **USB-CAN 串口转接器** 通信，而非直连 SocketCAN。

```
PC ──USB──► USB-CAN 转接器 ──CAN 总线──► RH56DFX 灵巧手
              (/dev/ttyUSB0)              (Hand ID 默认=1)
```

| 参数 | 默认值 |
|------|--------|
| 串口设备 | `/dev/ttyUSB0`（Linux） |
| 串口波特率 | **115200** |
| CAN 波特率 | 1000K（CAN 模式） |

PC 与转接器之间传输的是 **Serial-CAN 封装帧**（含帧头 `AA AA`、帧尾 `55 55`），转接器负责将其转换为 CAN 总线上的扩展帧。下文第 2 章描述该封装格式；第 3 章起描述 CAN 逻辑层（ext_id + data）。

## 2. Serial-CAN 串口封装协议

### 2.1 总体结构

Serial-CAN 帧在 CAN 逻辑帧外包一层串口协议，用于 USB-CAN 转接器通信。

**写入帧（主机 → 灵巧手）：**

```
┌────────┬──────────┬────────────┬─────────┬──────────────┬──────────┬──────┐
│ 帧头   │ ext_id   │ CAN 数据域 │ padding │ 固定字段     │ 校验和   │ 帧尾 │
│ AA AA  │ 4 字节   │ ≤8 字节    │ 0xFF 补 │ 长度+00+01+00│ 1 字节   │55 55 │
└────────┴──────────┴────────────┴─────────┴──────────────┴──────────┴──────┘
```

**读取帧（主机 → 灵巧手）：**

```
┌────────┬──────────┬────────────────────┬──────────────┬──────────┬──────┐
│ 帧头   │ ext_id   │ 8 字节数据区       │ 固定字段     │ 校验和   │ 帧尾 │
│ AA AA  │ 4 字节   │ [读长度,0,0,...]   │ 01 00 01 00  │ 1 字节   │55 55 │
└────────┴──────────┴────────────────────┴──────────────┴──────────┴──────┘
```

**回复帧（灵巧手 → 主机）：** 结构与请求帧相同（帧头 + ext_id + 8 字节数据区 + 固定字段 + 校验和 + 帧尾），数据域携带返回的寄存器值或写入确认。

### 2.2 ext_id 字节序

29 位扩展标识符在串口帧中占 **4 字节**，采用 **低字节在前（Little-Endian）**：

| 整型 CAN ID | 串口字节序 |
|-------------|-----------|
| `0x05738001` | `01 80 73 05` |
| `0x01840001` | `01 00 84 01` |
| `0x07EC0001` | `01 00 EC 07` |

转换公式：

```python
def can_id_to_bytes(can_id):
    can_id &= 0x1FFFFFFF
    return [
        can_id & 0xFF,
        (can_id >> 8) & 0xFF,
        (can_id >> 16) & 0xFF,
        (can_id >> 24) & 0xFF,
    ]
```

### 2.3 写入帧字段详解

| 偏移 | 长度 | 字段 | 说明 |
|------|------|------|------|
| 0 | 2 | 帧头 | 固定 `0xAA 0xAA` |
| 2 | 4 | ext_id | 29 位扩展标识符，低字节在前 |
| 6 | N | 数据 | 实际 CAN 数据，N ≤ 8 |
| 6+N | 8-N | padding | 不足 8 字节时补 `0xFF` |
| 14 | 1 | 数据长度 | 固定为 **8**（有效数据 + padding 总长） |
| 15 | 1 | 固定 | `0x00` |
| 16 | 1 | 固定 | `0x01` |
| 17 | 1 | 固定 | `0x00` |
| 18 | 1 | 校验和 | 见 2.5 |
| 19 | 2 | 帧尾 | 固定 `0x55 0x55` |

> **写入帧总长度**：2 + 4 + 8 + 4 + 1 + 2 = **21 字节**

### 2.4 读取帧字段详解

| 偏移 | 长度 | 字段 | 说明 |
|------|------|------|------|
| 0 | 2 | 帧头 | 固定 `0xAA 0xAA` |
| 2 | 4 | ext_id | 29 位扩展标识符，低字节在前 |
| 6 | 8 | 数据区 | 第 1 字节为预读长度，其余填 `0x00` |
| 14 | 1 | 固定 | `0x01` |
| 15 | 1 | 固定 | `0x00` |
| 16 | 1 | 固定 | `0x01` |
| 17 | 1 | 固定 | `0x00` |
| 18 | 1 | 校验和 | 见 2.5 |
| 19 | 2 | 帧尾 | 固定 `0x55 0x55` |

> **读取帧总长度**：**21 字节**

### 2.5 校验和计算

校验范围：从 **ext_id 第 1 字节** 到 **固定字段最后一字节**（不含帧头、不含校验和、不含帧尾）。

```python
checksum = sum(ext_id_bytes + data_field + meta_bytes) & 0xFF
```

**写入示例**（设置小拇指角度 500）：

```
ext_id  : 01 80 73 05
data    : F4 01
padding : FF FF FF FF FF FF
meta    : 08 00 01 00
checksum: (01+80+73+05+F4+01+FF*6+08+00+01+00) & 0xFF = 0xF1
```

完整写入帧：

```
AA AA 01 80 73 05 F4 01 FF FF FF FF FF FF 08 00 01 00 F1 55 55
```

### 2.6 回复帧解析

#### 写入回复

写入成功后，灵巧手返回确认帧，数据域通常为全零：

```
AA AA 01 80 73 05 08 00 00 00 00 00 00 00 01 00 01 00 03 55 55
              ↑ ext_id 与请求相同    ↑ 8 字节确认数据（全 0）
```

#### 读取回复

数据从偏移 **6** 开始，有效字节数由偏移 **14** 的字段指示（读 2 字节角度时该字段为 `0x02`）：

```
AA AA 01 00 84 01 C9 03 00 00 00 00 00 00 02 00 01 00 A5 55 55 55
              ↑ ext_id              ↑ 角度低 ↑ 角度高  ↑ 有效2字节
```

解析角度（低字节在前）：

```python
angle = resp[6] | (resp[7] << 8)   # 0x03C9 = 969
```

#### 0xA5 转义机制

当校验和或数据中恰好出现 `0x55`、`0xAA`、`0xA5` 时，转接器会用 `0xA5` 作为转义前缀，避免与帧尾 `55 55` 冲突：

| 原始字节 | 线上编码 |
|----------|----------|
| `0x55` | `A5 55` |
| `0xAA` | `A5 AA` |
| `0xA5` | `A5 A5` |

因此读回复末尾 `A5 55 55 55` 表示：校验和 = `0x55`（转义为 `A5 55`），帧尾 = `55 55`，**并非多收字节**。

解转义代码：

```python
def remove_a5_escape(data: bytes) -> bytes:
    out = bytearray()
    i = 0
    while i < len(data):
        if data[i] == 0xA5 and i + 1 < len(data) and data[i + 1] in (0x55, 0xAA, 0xA5):
            i += 1
            out.append(data[i])
        else:
            out.append(data[i])
        i += 1
    return bytes(out)
```

## 3. CAN 逻辑帧格式

### 3.1 帧类型要求
- ✅ 使用**扩展标识符**（29位）
- ✅ 使用**数据帧**
- ❌ 不使用标准标识符
- ❌ 不使用远程帧

### 3.2 扩展标识符位定义（29位，从低位至高位）

```
bit[31:29]  bit[28:26]  bit[25:14]  bit[13:0]
┌─────────┬───────────┬───────────┬──────────┐
│  预留   │  R/W标志  │  寄存器   │ Hand_ID  │
│  (3bit) │  (3bit)   │  地址     │ (14bit)  │
│   0x0   │           │ (12bit)   │          │
└─────────┴───────────┴───────────┴──────────┘
```

---

## 4. 扩展标识符定义

### 4.1 读写标志位（bit 26~28）

| 值 | 含义 | 说明 |
|----|------|------|
| `0` | 读灵巧手寄存器 | 读取手指相关寄存器 |
| `1` | 写灵巧手寄存器 | 写入手指相关寄存器 |
| `4` | 读灵巧手手腕寄存器 | 读取手腕相关寄存器 |
| `5` | 写灵巧手手腕寄存器 | 写入手腕相关寄存器 |

### 4.2 Hand_ID（bit 0~13）
- 范围：`0x0000` ~ `0x3FFF`
- 最大支持 **16383** 个设备
- 默认值：`0x01`

### 4.3 寄存器地址（bit 14~25）
- 12位地址，范围：`0x000` ~ `0xFFF`

### 4.4 ext_id 构建与解析

```python
def build_can_id(rw_flag, address, hand_id):
    can_id = 0
    can_id |= (rw_flag & 0x7) << 26    # bit 26-28: R/W 标志
    can_id |= (address & 0xFFF) << 14  # bit 14-25: 寄存器地址
    can_id |= (hand_id & 0x3FFF)       # bit 0-13:  Hand ID
    return can_id

def parse_can_id(can_id):
    rw_flag = (can_id >> 26) & 0x7
    address = (can_id >> 14) & 0xFFF
    hand_id = can_id & 0x3FFF
    return rw_flag, address, hand_id
```

## 5. 读写寄存器操作

### 5.1 读寄存器操作（R/W = 0）

#### 发送帧（主机 → 灵巧手）

| 字段 | 值 |
|------|-----|
| 标识符 | `0x000 | (0 << 26) | (Address << 14) | Hand_ID` |
| 数据长度 | 1 Byte |
| 数据内容 | 预读取寄存器数据的长度（字节数） |

#### 回复帧（灵巧手 → 主机）

| 字段 | 值 |
|------|-----|
| 标识符 | 与发送帧相同（R/W=0） |
| 数据长度 | 返回寄存器数据的长度 |
| 数据内容 | 寄存器数据（**低字节在前，高字节在后**） |

#### 读寄存器示例

**场景**：读取 ID=1 的灵巧手食指当前实际角度（`ANGLE_ACT(3)`，地址=1552）

```
CAN 逻辑层 - 发送：
  标识符：0x01840001
  解析：
    bit[28:26] = 000 (R/W=0, 读操作)
    bit[25:14] = 011000010000 (地址=1552=0x610)
    bit[13:0]  = 00000000000001 (Hand_ID=1)
  数据：[0x02]  (预读 2 字节)

Serial-CAN 读请求：
  AA AA 01 00 84 01 02 00 00 00 00 00 00 00 01 00 01 00 8A 55 55

CAN 逻辑层 - 回复：
  标识符：0x01840001
  数据：[0xC9, 0x03]  (实测值，低字节在前)

Serial-CAN 读回复（实测）：
  AA AA 01 00 84 01 C9 03 00 00 00 00 00 00 02 00 01 00 A5 55 55 55

  解析：0x03C9 = 969 → 食指当前实际角度为 969
```

---

### 5.2 写寄存器操作（R/W = 1）

#### 发送帧（主机 → 灵巧手）

| 字段 | 值 |
|------|-----|
| 标识符 | `0x000 | (1 << 26) | (Address << 14) | Hand_ID` |
| 数据长度 | 欲写入数据的长度（字节数） |
| 数据内容 | 欲写入的数据（**低字节在前，高字节在后**） |

#### 回复帧（灵巧手 → 主机）

| 字段 | 值 |
|------|-----|
| 标识符 | `0x000 | (1 << 26) | (Address << 14) | Hand_ID` |
| 数据长度 | 0（表示写入成功） |

#### 写寄存器示例

**场景**：将 ID=1 的灵巧手食指角度设置值设为 600（`ANGLE_SET(3)`，地址=1492）

```
CAN 逻辑层 - 发送：
  标识符：0x05750001  (R/W=1, Address=0x5D4=1492, Hand_ID=1)
  数据：[0x58, 0x02]   (600 = 0x0258，低字节在前)

Serial-CAN 写请求：
  AA AA 01 00 75 05 58 02 FF FF FF FF FF FF 08 00 01 00 [CS] 55 55

CAN 逻辑层 - 回复：
  标识符：0x05750001，数据域全零 → 写入成功
```

## 6. 常用寄存器地址表

**CAN模式：**
| 值 | 波特率 |
|----|--------|

| 值 | 仲裁段波特率 | 数据段波特率 |

- 写入 `1` 后，灵巧手清除可清除故障：

  - ✅ 异常故障
  - ✅ 通讯故障

  标识符：0x01840001  (R/W=0, Address=0x610=1552, Hand_ID=1)
  请求数据：[0x02]     (预读 2 字节)

Serial-CAN 读请求：
  AA AA 01 00 84 01 02 00 00 00 00 00 00 00 01 00 01 00 8A 55 55

Serial-CAN 读回复（实测）：
  AA AA 01 00 84 01 C9 03 00 00 00 00 00 00 02 00 01 00 A5 55 55 55
  → 食指实际角度 = 0x03C9 = 969
```

### 8.1 协议库（`1.py`）

项目中的 `1.py` 实现了完整的协议封装，主要 API：

| 函数/类 | 说明 |
|---------|------|
| `RH56DFXProtocol(hand_id)` | 协议主类，构建读写 CAN 逻辑帧 |

| `frame_to_tx_bytes_read()` | 读取组帧 |

| `short_to_bytes()` / `bytes_to_short()` | 角度值与字节互转 |

```

### 8.2 硬件调用（`2.py`）

`2.py` 演示如何通过串口连接实际设备：

```python
import serial
import time
from importlib import import_module

hand_mod = import_module("1")
hand = hand_mod.RH56DFXProtocol(hand_id=1)

ser = serial.Serial("/dev/ttyUSB0", 115200, timeout=0.5)

def send_frame(ser, frame, wait=0.05):
    ser.reset_input_buffer()
    tx = bytes(hand_mod.frame_to_tx_bytes(frame))
    ser.write(tx)
    ser.flush()
    time.sleep(wait)
    return ser.read(256)

# 写角度
send_frame(ser, hand.set_angle(hand_mod.FingerIndex.PINKY, 500))

# 读角度
resp = send_frame(ser, hand.read_angle_act(hand_mod.FingerIndex.INDEX))
angle = resp[6] | (resp[7] << 8)
print(f"食指实际角度: {angle}")

ser.close()
```

### 8.3 组帧流程

```
应用层调用                CAN 逻辑层              Serial-CAN 层
─────────────            ─────────────           ──────────────
hand.set_angle(500)  →  CANFrame(can_id,data) → frame_to_tx_bytes()
                                                    ↓
                                              ser.write(21字节)
                                                    ↓
                                              USB-CAN 转接器
                                                    ↓
                                              CAN 总线 → 灵巧手
```

### 8.4 注意事项

1. **串口权限**：Linux 下需将用户加入 `dialout` 组
2. **帧间延时**：连续发送建议间隔 10~50 ms
3. **批量写角度**：6 个手指需分 6 帧发送（每指独立地址）
4. **直连 CAN**：若使用 SocketCAN 而非串口转接器，只需发送 CAN 逻辑层（ext_id + data），**不需要** Serial-CAN 封装
