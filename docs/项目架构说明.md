# 灵巧手控制项目架构说明

## 一、项目概述

本项目是一个基于ROS2的灵巧手控制系统，支持通过485串口与多个灵巧手设备进行通信。系统采用模块化设计，支持多设备并行控制，提供话题（Topic）和服务（Service）两种通信方式。

### 1.1 核心特性

- ✅ **多设备支持**：支持同时控制多个灵巧手设备（如左手、右手）
- ✅ **多协议支持**：通过工厂模式支持多种通信协议（RH56F1_485、RH5DG2_485等）
- ✅ **动态配置**：通过YAML配置文件灵活配置设备和ROS2接口
- ✅ **双通信模式**：支持话题（实时控制）和服务（按需调用）两种方式
- ✅ **异步串口通信**：基于Boost.Asio的异步串口通信，支持超时和错误处理
- ✅ **统一日志系统**：全局日志管理器，支持文件轮转和级别控制
- ✅ **线程安全**：多线程环境下的安全通信

## 五、关键组件说明

### 5.1 硬件接口层

**功能**：与灵巧手硬件进行物理通信

**组件**：
- **485串口**：通过USB转485适配器连接设备
- **串口设备**：`/dev/ttyUSB0`、`/dev/ttyUSB1`等

**特点**：
- 支持多设备并行通信
- 每个设备独立串口和协议实例
- 异步接收，阻塞式发送

---

### 5.2 设备管理层

**功能**：管理串口设备和协议对象

**组件**：
- **SerialPortBase**：串口抽象基类
  - 基于Boost.Asio的异步通信
  - 支持超时读取
  - 线程安全的接收缓冲区
- **DeviceManager**：设备管理器
  - 管理多个串口设备
  - 端口到设备的映射
- **RingBuffer**：环形缓冲区
  - 缓存串口接收数据
  - 支持帧解析和数据提取

---

### 5.3 协议抽象层

**功能**：实现具体的通信协议

**组件**：
- **Protocol**：协议抽象基类
  - 定义统一的协议接口
  - 命令构建和响应解析
- **具体协议实现**：
  - `RH56F1_485_Protocol`：RH56F1系列协议
  - `RH5DG2_485_Protocol`：RH5DG2系列协议
- **ProtocolFactory**：协议工厂
  - 动态创建协议对象
  - 支持协议注册和扩展

**特点**：
- 工厂模式，易于扩展新协议
- 每个设备独立的协议实例
- 支持动态读取长度

---

### 5.4 ROS2接口层

**功能**：提供ROS2话题和服务接口

**组件**：
- **RegisterController**：设备控制节点
  - 一个设备对应一个节点
  - 支持多个话题和服务
  - 定时器循环读取和发布

**通信方式**：
1. **话题模式（Topic）**：
   - 订阅控制命令话题
   - 定时发布状态话题
   - 实时性高，适合连续控制

2. **服务模式（Service）**：
   - 提供与各寄存器对应的专用服务类型（由机型接口包定义）
   - 按需调用，不参与定时循环
   - 适合单次操作和查询

---

### 5.5 配置系统

**功能**：从YAML文件加载配置

**配置文件**：
1. **device_protocol_config.yaml**：
   - 设备配置（端口、波特率、Hand_ID）
   - 协议类型配置
   - 日志配置

2. **ros2_controller_config.yaml**：
   - ROS2节点配置
   - 话题配置（命令话题、状态话题）
   - 服务配置（Set/Get服务）

**组件**：
- **ConfigLoader**：配置加载器
  - 解析YAML文件
  - 创建协议对象
  - 配置日志系统

---

### 5.6 日志系统

**功能**：统一的日志管理

**组件**：
- **LoggerManager**：日志管理器
  - 全局单例模式
  - 基于spdlog
  - 支持控制台和文件输出
  - 支持日志轮转

**特点**：
- 线程安全
- 支持动态调整日志级别
- 文件自动轮转（按大小和数量）

## 六、线程模型

```
主线程 (inspire_control_node)
│
├─ ROS2事件循环线程 (MultiThreadedExecutor)
│  └─ RegisterController节点
│     ├─ 定时器线程 (50Hz)
│     │  └─ controlLoop(): 读取寄存器并发布
│     ├─ 话题订阅回调线程
│     │  └─ handleCommand(): 写入寄存器
│     └─ 服务回调线程
│        └─ InterfaceAdapter + rh*_interfaces：话题与服务类型绑定
│
└─ 串口异步接收线程 (每个设备一个)
   └─ SerialPortBase::io_thread_
      └─ Boost.Asio异步读取
         └─ 数据存入receive_buffer_
```

**线程安全机制**：
- 串口写入使用互斥锁保护
- 串口读取使用条件变量同步
- 定时器暂停机制防止服务与定时器冲突
- RingBuffer支持多线程访问

```
1. 程序启动 (inspire_control_node)
   │
   ├─ 2. 初始化ROS2 (rclcpp::init)
   │
```

## 八、扩展点

### 8.1 添加新协议

1. 创建新协议类，继承`Protocol`
2. 实现所有纯虚函数
3. 使用`REGISTER_PROTOCOL`宏注册
4. 在配置文件中指定协议类型

### 8.2 添加新寄存器

1. 在协议类的`REGISTER_MAP`中添加寄存器地址
2. 在`REGISTER_READ_LENGTH_MAP`中配置读取长度（如需要）
3. 在ROS2配置文件中添加话题或服务配置

### 8.3 添加新设备

1. 在`device_protocol_config.yaml`中添加设备配置
2. 在`ros2_controller_config.yaml`中添加设备节点配置
3. 系统自动识别并启动

## 九、性能特性

- **控制频率**：50Hz（20ms周期）
- **串口通信**：异步接收，阻塞式发送
- **超时机制**：读取超时25ms
- **多设备支持**：每个设备独立线程和协议实例
- **内存管理**：使用智能指针，自动管理资源

---

## 十、故障处理

- **串口通信错误**：超时重试，日志记录
- **协议解析错误**：跳过无效帧，继续处理
- **服务调用异常**：try-catch保护，确保响应返回
- **定时器冲突**：服务调用时暂停定时器3ms
