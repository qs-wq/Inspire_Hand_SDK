# RH5DG2_485 协议格式说明

## 一、协议概述

RH5DG2_485协议是一种基于485串口通信的协议，用于与RH5DG2设备进行数据交互。协议采用固定帧格式，包含帧头、设备ID、数据长度、命令、地址、数据和校验和等字段。

**重要说明**：
- 所有多字节数据采用**小端序**（Little-Endian）：低字节在前，高字节在后
- 地址采用16位，低字节在前，高字节在后
- 数据值采用16位，低字节在前，高字节在后
- 校验和计算范围：从`byte[2]`（设备ID）开始到倒数第二个字节（不包括校验和本身）

## 二、写寄存器请求格式（Write Request）

### 2.1 帧结构

| 字节位置 | 字段名称 | 长度 | 说明 | 示例值 |
|---------|---------|------|------|--------|
| byte[0] | 帧头低字节 | 1 | 固定值 `0xEB` | `0xEB` |
| byte[1] | 帧头高字节 | 1 | 固定值 `0x90` | `0x90` |
| byte[2] | 设备ID | 1 | 设备标识（Hand_ID） | `0x01` |
| byte[3] | 数据长度 | 1 | `值数量 × 2 + 3` | `0x23` (13个值) |
| byte[4] | 命令 | 1 | 固定值 `0x12`（写命令） | `0x12` |
| byte[5] | 地址低字节 | 1 | 寄存器地址的低8位 | `0x38` |
| byte[6] | 地址高字节 | 1 | 寄存器地址的高8位 | `0x04` |
| byte[7] ~ byte[6+2N] | 数据 | 2N | N个16位值，每个值2字节（低字节在前） | `0x00 0x00` |
| ... | ... | ... | ... | ... |
| byte[6+2N+1] | 校验和 | 1 | `byte[2]`到`byte[6+2N]`的累加和 | `0xXX` |

### 2.2 数据长度计算

```
数据长度 = 值数量 × 2 + 3
```

**说明**：
- `值数量 × 2`：每个值占2字节（16位）
- `+ 3`：命令(1字节) + 地址(2字节)

**示例**：
- 写入1个值：数据长度 = `1 × 2 + 3 = 5`
- 写入13个值：数据长度 = `13 × 2 + 3 = 29` (0x1D)

### 2.3 地址格式

地址采用16位，小端序：
- `byte[5]`：地址低字节（`address & 0xFF`）
- `byte[6]`：地址高字节（`(address >> 8) & 0xFF`）

**示例**：
- 地址 `0x0438` (1080)：
  - `byte[5] = 0x38`
  - `byte[6] = 0x04`

### 2.4 数据格式

每个值占2字节，小端序：
- 第一个字节：低字节（`value & 0xFF`）
- 第二个字节：高字节（`(value >> 8) & 0xFF`）

**示例**：
- 值 `0x1234`：
  - `byte[0] = 0x34`
  - `byte[1] = 0x12`

### 2.5 校验和计算

```cpp
校验和 = byte[2] + byte[3] + byte[4] + ... + byte[6+2N]
```

**说明**：
- 计算范围：从`byte[2]`（设备ID）开始到最后一个数据字节
- 不包括帧头和校验和本身
- 结果取低8位（`uint8_t`）

### 2.6 完整示例

**写入寄存器 `angleSet`（地址0x0438），值：[100, 200, 300, 400, 500, 600, 700, 800, 900, 1000, 1100, 1200, 1300]**

```
帧头：     0xEB 0x90
设备ID：   0x01
数据长度： 0x1D (13个值 × 2 + 3 = 29)
命令：     0x12
地址：     0x38 0x04 (0x0438，小端序)
数据：     0x64 0x00 (100)
           0xC8 0x00 (200)
           0x2C 0x01 (300)
           0x90 0x01 (400)
           0xF4 0x01 (500)
           0x58 0x02 (600)
           0xBC 0x02 (700)
           0x20 0x03 (800)
           0x84 0x03 (900)
           0xE8 0x03 (1000)
           0x4C 0x04 (1100)
           0xB0 0x04 (1200)
           0x14 0x05 (1300)
校验和：   0xXX (计算值)
```

**完整帧**：
```
0xEB 0x90 0x01 0x1D 0x12 0x38 0x04 0x64 0x00 0xC8 0x00 0x2C 0x01 0x90 0x01 0xF4 0x01 0x58 0x02 0xBC 0x02 0x20 0x03 0x84 0x03 0xE8 0x03 0x4C 0x04 0xB0 0x04 0x14 0x05 0xXX
```

## 三、读寄存器请求格式（Read Request）

### 3.1 帧结构

| 字节位置 | 字段名称 | 长度 | 说明 | 示例值 |
|---------|---------|------|------|--------|
| byte[0] | 帧头低字节 | 1 | 固定值 `0xEB` | `0xEB` |
| byte[1] | 帧头高字节 | 1 | 固定值 `0x90` | `0x90` |
| byte[2] | 设备ID | 1 | 设备标识（Hand_ID） | `0x01` |
| byte[3] | 数据长度 | 1 | 固定值 `0x04` | `0x04` |
| byte[4] | 命令 | 1 | 固定值 `0x11`（读命令） | `0x11` |
| byte[5] | 地址低字节 | 1 | 寄存器地址的低8位 | `0x70` |
| byte[6] | 地址高字节 | 1 | 寄存器地址的高8位 | `0x04` |
| byte[7] | 读取长度 | 1 | 期望读取的字节数 | `0x1A` (26字节) |
| byte[8] | 校验和 | 1 | `byte[2]`到`byte[7]`的累加和 | `0xXX` |

### 3.2 读取长度说明

`byte[7]`表示期望读取的**字节数**，不是值的数量。

**常见值**：
- `0x02`：读取1个16位值（2字节）
- `0x1A`：读取13个16位值（26字节）
- `0x44`：读取触觉数据（68字节）

### 3.3 校验和计算

```cpp
校验和 = byte[2] + byte[3] + byte[4] + byte[5] + byte[6] + byte[7]
```

### 3.4 完整示例

**读取寄存器 `angleAct`（地址0x0470），期望读取26字节（13个值）**

```
帧头：     0xEB 0x90
设备ID：   0x01
数据长度： 0x04 (固定)
命令：     0x11
地址：     0x70 0x04 (0x0470，小端序)
读取长度： 0x1A (26字节)
校验和：   0xXX (计算值)
```

**完整帧**：
```
0xEB 0x90 0x01 0x04 0x11 0x70 0x04 0x1A 0xXX
```

## 四、写寄存器回复格式（Write Response）

### 4.1 帧结构

| 字节位置 | 字段名称 | 长度 | 说明 | 示例值 |
|---------|---------|------|------|--------|
| byte[0] | 帧头低字节 | 1 | 固定值 `0x90` | `0x90` |
| byte[1] | 帧头高字节 | 1 | 固定值 `0xEB` | `0xEB` |
| byte[2] | 设备ID | 1 | 设备标识（Hand_ID） | `0x01` |
| byte[3] | 数据长度 | 1 | 固定值 `0x04` | `0x04` |
| byte[4] | 命令 | 1 | 固定值 `0x12`（写命令） | `0x12` |
| byte[5] | 地址低字节 | 1 | 寄存器地址的低8位 | `0x38` |
| byte[6] | 地址高字节 | 1 | 寄存器地址的高8位 | `0x04` |
| byte[7] | 固定值 | 1 | 固定值 `0x01`（表示写入成功） | `0x01` |
| byte[8] | 校验和 | 1 | `byte[2]`到`byte[7]`的累加和 | `0xXX` |

**注意**：回复帧的帧头顺序与请求帧相反（`0x90 0xEB` vs `0xEB 0x90`）

### 4.2 帧长度

写寄存器回复固定为**9字节**。

### 4.3 校验和计算

```cpp
校验和 = byte[2] + byte[3] + byte[4] + byte[5] + byte[6] + byte[7]
```

### 4.4 完整示例

**写入寄存器 `angleSet`（地址0x0438）的回复**

```
帧头：     0x90 0xEB
设备ID：   0x01
数据长度： 0x04
命令：     0x12
地址：     0x38 0x04
固定值：   0x01
校验和：   0xXX
```

**完整帧**：
```
0x90 0xEB 0x01 0x04 0x12 0x38 0x04 0x01 0xXX
```

## 五、读寄存器回复格式（Read Response）

### 5.1 帧结构

| 字节位置 | 字段名称 | 长度 | 说明 | 示例值 |
|---------|---------|------|------|--------|
| byte[0] | 帧头低字节 | 1 | 固定值 `0x90` | `0x90` |
| byte[1] | 帧头高字节 | 1 | 固定值 `0xEB` | `0xEB` |
| byte[2] | 设备ID | 1 | 设备标识（Hand_ID） | `0x01` |
| byte[3] | 数据长度 | 1 | `Register_Length + 3` | `0x1D` (26+3=29) |
| byte[4] | 命令 | 1 | 固定值 `0x11`（读命令） | `0x11` |
| byte[5] | 地址低字节 | 1 | 寄存器地址的低8位 | `0x70` |
| byte[6] | 地址高字节 | 1 | 寄存器地址的高8位 | `0x04` |
| byte[7] ~ byte[6+Register_Length] | 数据 | Register_Length | 寄存器数据（每个值2字节，小端序） | `0x64 0x00` |
| ... | ... | ... | ... | ... |
| byte[6+Register_Length+1] | 校验和 | 1 | `byte[2]`到`byte[6+Register_Length]`的累加和 | `0xXX` |

### 5.2 数据长度计算

```
byte[3] = Register_Length + 3
```

**说明**：
- `Register_Length`：实际返回的数据字节数
- `+ 3`：命令(1字节) + 地址(2字节)

**示例**：
- 返回26字节数据：`byte[3] = 26 + 3 = 29` (0x1D)
- 返回2字节数据：`byte[3] = 2 + 3 = 5` (0x05)

### 5.3 数据格式

数据从`byte[7]`开始，每个值占2字节，小端序：
- `byte[7]`：第一个值的低字节
- `byte[8]`：第一个值的高字节
- `byte[9]`：第二个值的低字节
- `byte[10]`：第二个值的高字节
- ...

**值解析**：
```cpp
value = byte[j] | (byte[j+1] << 8)
```

**有符号数处理**：
- 如果值 > 32767，则为负数：`value = value - 65536`

### 5.4 帧长度计算

```
总长度 = 8 + Register_Length
```

**说明**：
- `8`：帧头(2) + ID(1) + 长度(1) + 命令(1) + 地址(2) + 校验和(1)
- `Register_Length`：数据字节数

### 5.5 校验和计算

```cpp
校验和 = byte[2] + byte[3] + byte[4] + byte[5] + byte[6] + ... + byte[6+Register_Length]
```

### 5.6 完整示例

**读取寄存器 `angleAct`（地址0x0470），返回13个值：[100, 200, 300, 400, 500, 600, 700, 800, 900, 1000, 1100, 1200, 1300]**

```
帧头：     0x90 0xEB
设备ID：   0x01
数据长度： 0x1D (26+3=29)
命令：     0x11
地址：     0x70 0x04
数据：     0x64 0x00 (100)
           0xC8 0x00 (200)
           0x2C 0x01 (300)
           0x90 0x01 (400)
           0xF4 0x01 (500)
           0x58 0x02 (600)
           0xBC 0x02 (700)
           0x20 0x03 (800)
           0x84 0x03 (900)
           0xE8 0x03 (1000)
           0x4C 0x04 (1100)
           0xB0 0x04 (1200)
           0x14 0x05 (1300)
校验和：   0xXX
```

**完整帧**（34字节）：
```
0x90 0xEB 0x01 0x1D 0x11 0x70 0x04 0x64 0x00 0xC8 0x00 0x2C 0x01 0x90 0x01 0xF4 0x01 0x58 0x02 0xBC 0x02 0x20 0x03 0x84 0x03 0xE8 0x03 0x4C 0x04 0xB0 0x04 0x14 0x05 0xXX
```

## 六、触觉数据读取格式（Touch Data）

### 6.1 读取请求

触觉数据使用**读寄存器请求**格式，地址为`touchAct`（0x0BB8），读取长度为68字节。

**示例**：
```
0xEB 0x90 0x01 0x04 0x11 0xB8 0x0B 0x44 0xXX
```

### 6.2 读取回复

触觉数据使用**读寄存器回复**格式，返回68字节数据。

**数据格式**：
- **手指数据**（5个手指，每个10字节，共50字节）：
  - 每个手指包含：
    - 3个16位值（6字节）
    - 1个24位值（3字节）
    - 1字节保留（1字节）
- **掌心数据**（9个16位值，共18字节）

**完整帧长度**：
```
总长度 = 8 + 68 = 76 字节
```

**数据长度字段**：
```
byte[3] = 68 + 3 = 71 (0x47)
```

## 七、协议字段总结

### 7.1 帧头

| 方向 | 帧头 | 说明 |
|------|------|------|
| 请求 | `0xEB 0x90` | 写请求、读请求 |
| 回复 | `0x90 0xEB` | 写回复、读回复 |

### 7.2 命令码

| 命令码 | 说明 |
|--------|------|
| `0x11` | 读寄存器 |
| `0x12` | 写寄存器 |

### 7.3 数据长度字段（byte[3]）

| 帧类型 | 计算公式 | 说明 |
|--------|---------|------|
| 写请求 | `值数量 × 2 + 3` | 值数量×2 + 命令(1) + 地址(2) |
| 读请求 | `0x04`（固定） | 固定值 |
| 写回复 | `0x04`（固定） | 固定值 |
| 读回复 | `Register_Length + 3` | 数据字节数 + 命令(1) + 地址(2) |

### 7.4 校验和计算范围

| 帧类型 | 计算范围 | 说明 |
|--------|---------|------|
| 写请求 | `byte[2]` ~ `byte[6+2N]` | 从设备ID到最后一个数据字节 |
| 读请求 | `byte[2]` ~ `byte[7]` | 从设备ID到读取长度 |
| 写回复 | `byte[2]` ~ `byte[7]` | 从设备ID到固定值 |
| 读回复 | `byte[2]` ~ `byte[6+Register_Length]` | 从设备ID到最后一个数据字节 |

## 八、字节序说明

### 8.1 小端序（Little-Endian）

所有多字节数据采用**小端序**（低字节在前，高字节在后）：

**16位值示例**：
- 值 `0x1234`：
  - 低字节：`0x34`
  - 高字节：`0x12`
  - 字节序列：`0x34 0x12`

**地址示例**：
- 地址 `0x0438`：
  - 低字节：`0x38`
  - 高字节：`0x04`
  - 字节序列：`0x38 0x04`

### 8.2 值解析

**16位值解析**：
```cpp
value = byte[0] | (byte[1] << 8)
```

**有符号数处理**：
```cpp
if (value > 32767) {
    value = value - 65536;  // 转换为负数
}
```

## 十一、注意事项

1. **帧头顺序**：请求帧头为`0xEB 0x90`，回复帧头为`0x90 0xEB`（顺序相反）
2. **数据长度**：写请求和读回复的`byte[3]`含义不同，需注意区分
3. **字节序**：所有多字节数据采用小端序（低字节在前）
4. **校验和**：计算范围从`byte[2]`开始，不包括帧头和校验和本身
5. **有符号数**：16位值如果>32767，需要转换为负数（减去65536）
6. **动态长度**：读回复的数据长度由`byte[3]`动态确定，需根据协议规范计算
7. **特殊处理**：
   - 写入 `defaultSpeedSet` 或 `defaultForceSet` 后，会自动写入 `save=1`
   - 写入 `actionSeqIndex` 后，会自动写入 `actionSeqRun=1`
   - 写入 `actionLibraryIndex` 后，会自动写入 `actionLibraryRun=1`
8. **寄存器特点**：
   - RH5DG2设备支持13个自由度（26字节），而RH56F1设备支持6个自由度（12字节）
   - 多值寄存器默认读取长度为26字节（13个值）
