## 因时机器人EG-5CD1电动夹爪 - 485通信编程指南

---

### 一、通信基础配置

| 参数 | 值 |
|------|-----|
| 接口类型 | RS485 |
| 波特率 | 115200（默认）/ 460800 / 921600 |
| 数据位 | 8位 |
| 停止位 | 1位 |
| 校验 | 无 |
| 帧格式 | 字节流，大端模式（Big Endian） |
| 设备ID范围 | 1-255（出厂默认ID=1） |

### 二、485自定义协议帧格式

#### 2.1 指令帧结构（主控→夹爪）

| 字节 | 字段 | 说明 |
|------|------|------|
| byte[0] | 包头1 | 固定 `0xEB` |
| byte[1] | 包头2 | 固定 `0x90` |
| byte[2] | Hands_ID | 夹爪ID号（1-255） |
| byte[3] | 长度 | 数据部分长度（不含包头和校验和） |
| byte[4] | 命令类型 | `0x00`=读寄存器 / `0x01`=写寄存器 |
| byte[5] | Address_L | 寄存器起始地址低8位 |
| byte[6] | Address_H | 寄存器起始地址高8位 |
| byte[7] | 数据/长度 | 读：寄存器长度(字节) / 写：数据开始 |
| byte[7+n] | Data[n] | 写操作时的数据（大端模式） |
| 最后 | Checksum | 校验和（除应答帧头外其余数据的累加和低字节）|

**校验和计算**：`Checksum = (byte[2] + byte[3] + ... + byte[n]) & 0xFF`

#### 2.2 应答帧结构（夹爪→主控）

| 字节 | 字段 | 说明 |
|------|------|------|
| byte[0] | 包头1 | 固定 `0xEE` |
| byte[1] | 包头2 | 固定 `0x16` |
| byte[2] | Hands_ID | 夹爪ID号 |
| byte[3] | 长度 | 数据部分长度 |
| byte[4] | 命令类型 | `0x00`=读回复 / `0x01`=写回复 |
| byte[5] | Address_L | 寄存器地址低8位 |
| byte[6] | Address_H | 寄存器地址高8位 |
| byte[7] | Data[0] | 数据开始（读操作）或确认码（写操作返回1） |
| ... | ... | ... |
| 最后 | Checksum | 校验和 |

### 四、故障码详解（ERROR_CODE - 地址1128）

| 位 | 名称 | 说明 | 可清除 |
|----|------|------|--------|
| Bit0 | 堵转故障 | 电机堵转 | ✅ 通过CLEAR_ERROR |
| Bit1 | 过温故障 | 温度>80℃触发，需降至<60℃自动恢复 | ❌ 只能等待降温 |
| Bit2 | 过流故障 | 电流异常 | ✅ 通过CLEAR_ERROR |
| Bit3 | 电机异常 | 电机故障 | ✅ 通过CLEAR_ERROR |
| Bit4 | 通讯故障 | 通信异常 | ✅ 通过CLEAR_ERROR |

### 五、状态码详解（STATUS - 地址1130）

| 状态码 | 含义 |
|--------|------|
| 0x01 | 夹爪张开到最大且空闲 |
| 0x02 | 夹爪闭合到最小且空闲 |
| 0x03 | 夹爪停止且空闲 |
| 0x04 | 夹爪正在闭合 |
| 0x05 | 夹爪正在张开 |
| 0x06 | 闭合时力控停止 |
| 0x07 | 松开时力控停止 |
| 0x08 | 触控模式抓取中 |
| 0x09 | 触控模式松开中 |

### 六、三种工作模式编程指南

#### 6.1 位置模式（默认模式，CATCH_MODE_SET=0）

**工作流程**：
1. 设置速度（SPEED_SET，0-1000）
2. 设置力（FORCE_SET，0-2000，正值）
3. 设置目标开口度（OPEN_LEN_SET，0-1000）
4. 夹爪自动运动到目标位置

**特性**：
- 力分为夹持力（内夹）和外撑力（外撑）
- 设定力时无论内夹外撑都设正值
- 读取FORCE_ACT时：正值=夹持力，负值=外撑力
- 运动中受力超过设定力则停止，需重新下发指令才继续

#### 6.2 力控模式（CATCH_MODE_SET=1）

**工作流程**：
1. 切换模式：写1010寄存器为1
2. 设置速度阈值（SPEED_SET，0-1000）— 非实际速度
3. 设置力（FORCE_SET，-2000~2000）：正值=夹取方向，负值=张开方向
4. 夹爪自动向最大或最小开口度运动

**特性**：
- 实际速度与设定力大小正相关（力越大越快）
- 无需设置开口度
- 遇到阻力达到设定力时保持恒力
- 撤去外力后继续向极限位置运动

**⚠️ 注意**：必须在模式切换完成后再设定速度！

#### 6.3 触控模式（CATCH_MODE_SET=2）

**工作流程**：
1. 切换模式：写1010寄存器为2
2. 设置速度（SPEED_SET，0-1000）
3. 设置力（FORCE_SET，0-2000）
4. 控制夹取：写1136寄存器为1（CATCH_MODE_CLOSE）
5. 控制张开：写1138寄存器为1（CATCH_MODE_OPEN）

**特性**：
- 使用触觉传感器检测物体接近，自动减速
- 夹到物体后检测切向力变化（滑动趋势）
- 有滑动趋势时自动增大夹持力直至稳定
- 适合抓取易碎或易滑物体

### 七、编程示例代码框架

#### 7.1 校验和计算函数
```python
def calc_checksum(data_bytes):
    """计算485协议校验和"""
    return sum(data_bytes) & 0xFF
```

#### 7.2 读寄存器指令构造
```python
def build_read_frame(hand_id, address, length):
    """
    构造读寄存器指令帧
    hand_id: 夹爪ID (1-255)
    address: 寄存器地址 (如 0x0462)
    length: 读取字节数 (通常2)
    """
    addr_h = (address >> 8) & 0xFF
    addr_l = address & 0xFF
    
    frame = [
        0xEB, 0x90,           # 包头
        hand_id,              # ID
        0x04,                 # 数据长度
        0x00,                 # 读命令
        addr_l, addr_h,       # 地址低高字节
        length                # 读取长度
    ]
    frame.append(calc_checksum(frame[2:]))  # 校验和
    return bytes(frame)
```

#### 7.3 写寄存器指令构造（16位数据）
```python
def build_write_frame_16bit(hand_id, address, value):
    """
    构造写16位寄存器指令帧（大端模式）
    value: 0-65535，自动拆分为高字节+低字节
    """
    addr_h = (address >> 8) & 0xFF
    addr_l = address & 0xFF
    val_h = (value >> 8) & 0xFF
    val_l = value & 0xFF
    
    frame = [
        0xEB, 0x90,           # 包头
        hand_id,              # ID
        0x05,                 # 数据长度 (2字节数据+3字节地址命令)
        0x01,                 # 写命令
        addr_l, addr_h,       # 地址低高字节
        val_l, val_h          # 数据低高字节（大端模式！）
    ]
    frame.append(calc_checksum(frame[2:]))
    return bytes(frame)
```

#### 7.4 完整操作示例：设置开口度为1000
```python
# 设置ID为1的夹爪开口度为1000
# 寄存器地址：1020 (0x03FC)
# 数值：1000 = 0x03E8 → 低字节0xE8, 高字节0x03

frame = build_write_frame_16bit(1, 0x03FC, 1000)
# 发送帧: EB 90 01 05 01 FC 03 E8 03 [Checksum]

# 期望应答: EE 16 01 04 01 FC 03 01 [Checksum]
# 表示写入成功
```

### 八、关键编程注意事项

1. **字节序**：所有16位数据采用**大端模式**（高字节在前，低字节在后），但485协议帧中地址和数据部分在传输时是**低字节在前，高字节在后**（见示例）

2. **地址编码**：帧中地址字段是**低8位在前，高8位在后**（Little Endian传输），与数值存储的大端模式不同

3. **模式切换顺序**：力控/触控模式下，必须先切换模式，再设置速度，最后设置力

4. **故障处理**：
   - 过温故障（Bit1）不能通过CLEAR_ERROR清除
   - 重复出现的故障需要工程师维修

5. **参数保存**：SAVE和DEFAUL_PAR寄存器操作后立即生效，但写入的是触发指令，不是持久值

6. **多设备管理**：同一485总线最多支持254个设备，确保ID唯一

### 九、Modbus RTU协议支持（备用）

夹爪同时支持标准Modbus RTU协议，功能码：
- `0x03`：读取保持寄存器
- `0x06`：预置单寄存器  
- `0x10`：预置多寄存器

寄存器地址与自定义协议相同，但需使用Modbus CRC16校验。
