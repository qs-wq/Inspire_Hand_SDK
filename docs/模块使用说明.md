## 1. 串口通信模块 (SerialPortBase)

### 1.1 功能简介

`SerialPortBase`是串口通信的抽象基类，基于Boost.Asio实现异步串口通信。提供阻塞式读写接口，支持超时机制，内部使用异步接收线程持续接收数据。

### 1.2 主要类和函数

#### 类定义

```cpp
class SerialPortBase {
public:
    SerialPortBase(const std::string& port, unsigned int baudrate);
    ~SerialPortBase();
    
    // 阻塞式写入
    size_t write(const std::vector<uint8_t>& data);
    
    // 阻塞式读取（带超时）
    std::vector<uint8_t> read(std::chrono::milliseconds timeout = std::chrono::milliseconds(25));
    
    // 非阻塞式读取（返回当前缓冲区数据）
    std::vector<uint8_t> readAvailable();
    
    // 清空接收缓冲区
    void clearBuffer();
    
    // 检查串口状态
    bool isOpen() const;
};
```

### 1.3 配置参数

**构造函数参数**：
- `port`：串口设备路径（如`/dev/ttyUSB0`）
- `baudrate`：波特率（如115200）

**内部参数**：
- `BUFFER_SIZE = 4096`：接收缓冲区大小

### 1.4 使用示例

```cpp
#include "serial_port.hpp"

// 创建串口对象
auto serial = std::make_shared<SerialPortBase>("/dev/ttyUSB0", 115200);

// 检查串口是否打开
if (!serial->isOpen()) {
    std::cerr << "串口打开失败" << std::endl;
    return;
}

// 写入数据
std::vector<uint8_t> cmd = {0xEB, 0x90, 0x01, 0x04, 0x11, 0x28, 0x04, 0x0C, 0xXX};
size_t written = serial->write(cmd);

// 读取数据（阻塞，超时25ms）
auto data = serial->read(std::chrono::milliseconds(25));

// 非阻塞读取（立即返回当前缓冲区数据）
auto available_data = serial->readAvailable();

// 清空接收缓冲区
serial->clearBuffer();
```

### 1.5 数据流和通信方式

**写入流程**：
```
调用write() → Boost.Asio同步写入 → 串口硬件
```

**读取流程**：
```
异步接收线程 → 持续接收数据 → 存入receive_buffer_
调用read() → 从receive_buffer_读取 → 返回数据
```

**线程模型**：
- 主线程：调用`write()`和`read()`
- 异步线程：持续接收串口数据，存入缓冲区

## 2. 协议抽象层 (Protocol)

### 2.1 功能简介

`Protocol`是协议抽象基类，定义所有协议实现必须遵循的接口规范。具体协议（如`RH56F1_485_Protocol`）继承此类并实现协议特定的命令构建和响应解析逻辑。

### 2.2 主要类和函数

#### 基类接口

```cpp
class Protocol {
public:
    // 设置/获取设备ID
    void setDeviceId(uint8_t id);
    uint8_t getDeviceId() const;
    
    // 获取寄存器地址
    virtual int getRegisterAddress(const std::string& register_name) const = 0;
    
    // 构建命令
    virtual std::vector<uint8_t> buildReadCommand(int address, size_t length) = 0;
    virtual std::vector<uint8_t> buildWriteCommand(int address, const std::vector<int>& values) = 0;
    
    // 解析响应
    virtual std::pair<bool, std::vector<int>> parseResponse(RingBuffer& ringBuffer) = 0;
    virtual bool validateChecksum(const std::vector<uint8_t>& response) const = 0;
    
    // 触觉数据
    virtual std::pair<bool, TouchDataResult> parseTouchData(RingBuffer& ringBuffer, int version) = 0;
    
    // 高级接口
    virtual bool writeRegister(Device device, const std::string& reg_name, const std::vector<int>& values) = 0;
    virtual std::pair<bool, std::vector<int>> readRegister(Device device, RingBuffer& ringBuffer, const std::string& reg_name, size_t length) = 0;
    virtual std::pair<bool, TouchDataResult> readTouchData(Device device, RingBuffer& ringBuffer, int version) = 0;
};
```

#### 具体协议实现

**RH56F1_485_Protocol**：
- 实现RH56F1系列设备的485通信协议
- 支持动态读取长度
- 支持触觉数据解析（版本1和2）

**RH5DG2_485_Protocol**：
- 实现RH5DG2系列设备的485通信协议
- 与RH56F1协议类似，但寄存器地址不同
- 支持`actionLibraryIndex`特殊处理

### 2.3 配置参数

**设备ID（Hand_ID）**：
- 范围：1-254
- 默认：1
- 用途：区分同一串口总线上的多个设备

**寄存器映射**：
- 每个协议类维护一个`REGISTER_MAP`，将寄存器名称映射到地址
- 例如：`{"angleSet": 1040, "angleAct": 1064}`

**读取长度映射**：
- `REGISTER_READ_LENGTH_MAP`：定义每个寄存器的默认读取长度
- 例如：`{"angleAct": 12, "errorCode": 2}`

### 2.4 使用示例

```cpp
#include "RH56F1_485_protocol.hpp"
#include "serial_port.hpp"
#include "ring_buffer.hpp"

// 创建协议对象
auto protocol = std::make_shared<RH56F1_485_Protocol>();

// 设置设备ID
protocol->setDeviceId(1);

// 创建串口和缓冲区
auto device = std::make_shared<SerialPortBase>("/dev/ttyUSB0", 115200);
RingBuffer ringBuffer(1024);

// 写入寄存器
std::vector<int> values = {100, 200, 300, 400, 500, 600};
bool success = protocol->writeRegister(device, "angleSet", values);

// 读取寄存器（length=0表示自动确定长度）
auto [success, values] = protocol->readRegister(device, ringBuffer, "angleAct", 0);

// 读取触觉数据
auto [success, touchData] = protocol->readTouchData(device, ringBuffer, 1);
```

### 2.5 数据流和通信方式

**写入流程**：
```
writeRegister() 
  → buildWriteCommand() (构建命令帧)
  → device->write() (发送到串口)
  → readResponseWithLoop() (读取回复)
  → parseResponse() (解析回复)
  → 返回成功/失败
```

**读取流程**：
```
readRegister() 
  → buildReadCommand() (构建命令帧)
  → device->write() (发送到串口)
  → readResponseWithLoop() (读取回复)
  → parseResponse() (解析回复)
  → 返回值和成功标志
```

**帧格式**：
- 请求帧：`0xEB 0x90` + 设备ID + 数据长度 + 命令 + 地址 + 数据 + 校验和
- 回复帧：`0x90 0xEB` + 设备ID + 数据长度 + 命令 + 地址 + 数据 + 校验和

## 3. 设备管理器 (DeviceManager)

### 3.1 功能简介

`DeviceManager`管理多个串口设备，维护端口到设备对象的映射关系。每个设备对应一个`SerialPortBase`对象。

### 3.2 主要类和函数

```cpp
class DeviceManager {
public:
    // 添加设备
    void addDevice(const std::string& port, std::shared_ptr<Protocol> protocol, int baudRate);
    
    // 获取设备
    std::shared_ptr<SerialPortBase> getDevice(const std::string& port);
    
    // 移除设备
    void removeDevice(const std::string& port);
};
```

### 3.3 配置参数

**addDevice参数**：
- `port`：串口设备路径（如`/dev/ttyUSB0`）
- `protocol`：协议对象指针（用于设置设备协议）
- `baudRate`：波特率（如115200）

### 3.4 使用示例

```cpp
#include "device_manager.hpp"
#include "RH56F1_485_protocol.hpp"

// 创建设备管理器
DeviceManager device_manager;

// 创建协议对象
auto protocol = std::make_shared<RH56F1_485_Protocol>();
protocol->setDeviceId(1);

// 添加设备
device_manager.addDevice("/dev/ttyUSB0", protocol, 115200);

// 获取设备
auto device = device_manager.getDevice("/dev/ttyUSB0");
if (device) {
    // 使用设备
    std::vector<uint8_t> cmd = {0xEB, 0x90, 0x01, 0x04, 0x11, 0x28, 0x04, 0x0C, 0xXX};
    device->write(cmd);
}

// 移除设备
device_manager.removeDevice("/dev/ttyUSB0");
```

### 3.5 数据流和通信方式

**设备管理流程**：
```
addDevice() 
  → 创建SerialPortBase对象
  → 设置协议
  → 存入devices映射表
  → 启动异步接收线程
```

**设备获取流程**：
```
getDevice(port) 
  → 从devices映射表查找
  → 返回SerialPortBase对象
```

## 4. 配置加载器 (ConfigLoader)

### 4.1 功能简介

`ConfigLoader`负责从YAML配置文件加载设备配置、创建协议对象和配置日志系统。

### 4.2 主要类和函数

```cpp
class ConfigLoader {
public:
    // 加载设备配置
    static std::unordered_map<std::string, DeviceInfo> loadDeviceConfig(const std::string& config_path);
    
    // 创建协议对象
    static std::shared_ptr<Protocol> createProtocolFromConfig(const std::string& config_path);
    
    // 配置日志系统
    static void configureLogging(const std::string& config_path);
};
```

### 4.3 配置参数

**配置文件格式 (device_protocol_config.yaml)**：

```yaml
protocol:
  type: RH56F1_485  # 协议类型

devices:
  - name: hand_left      # 设备名称
    port: /dev/ttyUSB0   # 串口端口
    baudrate: 115200     # 波特率
    Hand_ID: 1           # 设备ID

logging:
  level: DEBUG           # 日志级别
  file: logs/hand_control.log  # 日志文件路径
  console: true          # 是否输出到控制台
  file_enable: true      # 是否输出到文件
  max_file_size_mb: 10   # 单个日志文件最大大小（MB）
  max_files: 5           # 保留的日志文件数量
```

### 4.4 使用示例

```cpp
#include "config_loader.hpp"
#include "device_manager.hpp"

// 配置日志系统
ConfigLoader::configureLogging("config/device_protocol_config.yaml");

// 加载设备配置
auto deviceConfig = ConfigLoader::loadDeviceConfig("config/device_protocol_config.yaml");

// 遍历设备配置
for (const auto& [port, deviceInfo] : deviceConfig) {
    std::cout << "设备: " << deviceInfo.name 
              << ", 端口: " << port 
              << ", 波特率: " << deviceInfo.baudrate 
              << ", Hand_ID: " << deviceInfo.hand_id << std::endl;
}

// 创建协议对象
auto protocol = ConfigLoader::createProtocolFromConfig("config/device_protocol_config.yaml");
```

### 4.5 数据流和通信方式

**配置加载流程**：
```
loadDeviceConfig() 
  → 读取YAML文件
  → 解析设备列表
  → 返回DeviceInfo映射表
```

**协议创建流程**：
```
createProtocolFromConfig() 
  → 读取YAML文件
  → 获取协议类型
  → ProtocolFactory::create()
  → 返回协议对象
```

## 5. 日志管理器 (LoggerManager)

### 5.1 功能简介

`LoggerManager`提供统一的日志管理功能，基于spdlog实现。支持控制台和文件输出，支持日志轮转和级别控制。

### 5.2 主要类和函数

```cpp
class LoggerManager {
public:
    // 初始化日志系统（从配置结构）
    static void initialize(const LogConfig& config);
    
    // 初始化日志系统（从YAML节点）
    static void initialize(const YAML::Node& logging_node);
    
    // 动态设置日志级别
    static void setLogLevel(const std::string& level);
    static void setLogLevel(spdlog::level::level_enum level);
    
    // 获取日志级别
    static std::string getLogLevel();
    
    // 获取日志对象
    static std::shared_ptr<spdlog::logger> getLogger();
    
    // 重新配置日志系统
    static void reconfigure(const LogConfig& config);
    
    // 刷新日志缓冲区
    static void flush();
    
    // 关闭日志系统
    static void shutdown();
};
```

### 5.3 配置参数

**LogConfig结构**：
```cpp
struct LogConfig {
    std::string level = "INFO";              // 日志级别
    std::string file_path = "logs/app.log";  // 日志文件路径
    bool console_enable = true;              // 是否输出到控制台
    bool file_enable = true;                 // 是否输出到文件
    size_t max_file_size = 10 * 1024 * 1024; // 单个日志文件最大大小（字节）
    size_t max_files = 5;                    // 保留的日志文件数量
};
```

**日志级别**：
- `TRACE`：最详细的日志
- `DEBUG`：调试信息
- `INFO`：一般信息
- `WARN`：警告信息
- `ERROR`：错误信息
- `CRITICAL`：严重错误
- `OFF`：关闭日志

### 5.4 使用示例

```cpp
#include "logger_manager.hpp"

// 方式1：从YAML配置初始化
YAML::Node config = YAML::LoadFile("config/device_protocol_config.yaml");
LoggerManager::initialize(config["logging"]);

// 方式2：从配置结构初始化
LoggerManager::LogConfig log_config;
log_config.level = "DEBUG";
log_config.file_path = "logs/app.log";
log_config.console_enable = true;
log_config.file_enable = true;
log_config.max_file_size = 10 * 1024 * 1024;
log_config.max_files = 5;
LoggerManager::initialize(log_config);

// 获取日志对象并使用
auto logger = getLogger();
logger->info("这是一条信息日志");
logger->debug("这是一条调试日志");
logger->warn("这是一条警告日志");
logger->error("这是一条错误日志");

// 动态调整日志级别
LoggerManager::setLogLevel("WARN");

// 刷新日志缓冲区
LoggerManager::flush();

// 关闭日志系统（程序退出时）
LoggerManager::shutdown();
```

### 5.5 数据流和通信方式

**日志输出流程**：
```
logger->info() 
  → spdlog日志系统
  → 控制台sink（如果启用）
  → 文件sink（如果启用）
  → 日志文件（自动轮转）
```

**日志轮转**：
- 当日志文件达到`max_file_size`时，自动创建新文件
- 保留`max_files`个日志文件，超过后删除最旧的

## 6. 环形缓冲区 (RingBuffer)

### 6.1 功能简介

`RingBuffer`是一个环形缓冲区，用于缓存串口接收的数据。支持高效的数据入队和出队操作，支持从指定位置读取数据而不破坏环形结构。

### 6.2 主要类和函数

```cpp
class RingBuffer {
public:
    explicit RingBuffer(size_t size);
    
    // 清除缓冲区
    void clear();
    
    // 入队，将数据添加到环形缓冲区
    void push(const uint8_t* data, size_t len);
    
    // 出队，从缓冲区读取数据
    size_t pop(uint8_t* data, size_t maxlen);
    
    // 获取当前缓冲区中的有效数据长度
    size_t size() const;
    
    // 获取从tail开始的连续数据长度
    size_t contiguousDataSize() const;
    
    // 获取缓冲区的底层数据指针
    const uint8_t* data() const;
    
    // 获取从tail开始的连续区段的指针
    const uint8_t* dataPtr() const;
    
    // 裁剪已解析的数据，移动tail指针
    void advance(size_t count);
    
    // 获取buffer底层数据（常量引用）
    const std::vector<uint8_t>& getBuffer() const;
    
    // 获取tail索引
    size_t getTail() const;
};
```

### 6.3 配置参数

**构造函数参数**：
- `size`：缓冲区大小（如1024）

### 6.4 使用示例

```cpp
#include "ring_buffer.hpp"

// 创建环形缓冲区
RingBuffer ringBuffer(1024);

// 添加数据
uint8_t data[] = {0x90, 0xEB, 0x01, 0x0F, 0x11, 0x28, 0x04, 0x64, 0x00};
ringBuffer.push(data, sizeof(data));

// 获取数据长度
size_t data_size = ringBuffer.size();

// 读取数据（不破坏环形结构）
uint8_t read_buffer[1024];
size_t read_len = ringBuffer.pop(read_buffer, sizeof(read_buffer));

// 从指定位置读取（用于协议解析）
size_t offset = 0;
uint8_t byte = readByteAtOffset(ringBuffer, offset);

// 裁剪已解析的数据
ringBuffer.advance(10);  // 移动tail指针，丢弃前10字节

// 清空缓冲区
ringBuffer.clear();
```

### 6.5 数据流和通信方式

**数据入队流程**：
```
push(data, len) 
  → 检查缓冲区空间
  → 写入数据到head位置
  → 更新head指针
```

**数据出队流程**：
```
pop(data, maxlen) 
  → 从tail位置读取数据
  → 更新tail指针
  → 返回读取长度
```

**数据裁剪流程**：
```
advance(count) 
  → 移动tail指针
  → 释放已解析的数据空间
```

## 7. 协议工厂 (ProtocolFactory)

### 7.1 功能简介

`ProtocolFactory`使用工厂模式和注册机制，支持动态注册和创建协议对象。新协议只需要注册即可，无需修改核心代码。

### 7.2 主要类和函数

```cpp
class ProtocolFactory {
public:
    // 注册协议类型
    static void registerProtocol(const std::string& type, ProtocolCreator creator);
    
    // 创建协议对象
    static std::shared_ptr<Protocol> create(const std::string& type);
    
    // 获取已注册的协议类型列表
    static std::vector<std::string> getRegisteredTypes();
    
    // 检查协议类型是否已注册
    static bool isRegistered(const std::string& type);
};
```

### 7.3 配置参数

**协议注册宏**：
```cpp
REGISTER_PROTOCOL("RH56F1_485", RH56F1_485_Protocol);
```

### 7.4 使用示例

```cpp
#include "protocol_factory.hpp"

// 检查协议是否已注册
if (ProtocolFactory::isRegistered("RH56F1_485")) {
    // 创建协议对象
    auto protocol = ProtocolFactory::create("RH56F1_485");
    
    // 使用协议
    protocol->setDeviceId(1);
}

// 获取所有已注册的协议类型
auto types = ProtocolFactory::getRegisteredTypes();
for (const auto& type : types) {
    std::cout << "已注册协议: " << type << std::endl;
}
```

### 7.5 数据流和通信方式

**协议注册流程**（程序启动时自动执行）：
```
REGISTER_PROTOCOL宏
  → 静态全局对象构造
  → 调用registerProtocol()
  → 存入注册表
```

**协议创建流程**：
```
create(type) 
  → 从注册表查找创建函数
  → 调用创建函数
  → 返回协议对象
```

## 8. ROS2控制器 (RegisterController)

### 8.1 功能简介

`RegisterController` 是 ROS2 设备控制节点，一个设备对应一个节点。对外 ROS 类型由 **`device_protocol_config.yaml`** 的 **`protocol.type`** 自动推导为 **`RH5DG2` / `RH56F1`**，经 **`InterfaceAdapter`** 映射到 **`rh5dg2_interfaces` 或 `rh56f1_interfaces`**（不再有统一的 `RegisterData` / `SetRegister`）。支持话题与服务，定时器循环读寄存器并发布状态。

### 8.2 主要类和函数

```cpp
class RegisterController : public rclcpp::Node {
public:
    RegisterController(
        const std::string& node_name,
        const DeviceNodeConfig& config,
        std::shared_ptr<SerialPortBase> device,
        std::shared_ptr<Protocol> protocol
    );
    
    // 初始化控制器
    void initialize();
    
    // 启动控制循环
    void start();
    
    // 停止控制循环
    void stop();
    
protected:
    // 读取寄存器值
    std::pair<bool, std::vector<int>> readRegister(const std::string& reg_name, size_t length = 0);
    
    // 写入寄存器值
    bool writeRegister(const std::string& reg_name, const std::vector<int>& values);
    
    // 读取触觉数据
    std::pair<bool, TouchDataResult> readTouchData(int version = 0);
    
    // 控制循环（定时器调用）
    void controlLoop();
};
```

### 8.3 配置参数

**DeviceNodeConfig结构**：
```cpp
struct DeviceNodeConfig {
    std::string device_name;              // 设备名称
    std::string interfaces_profile;       // 由 protocol.type 推导（RH5DG2 / RH56F1），勿在 ros2_controller_config.yaml 中配置
    std::string publish_frame_id;         // 来自 publish_header.frame_id 或 frame_id，写入发布消息的 Header
    std::vector<std::string> joint_names; // 与机型自由度一致（RH5DG2:13 / RH56F1:6），写入 joint_names 数组
    std::vector<TopicConfig> topics;      // 话题配置列表
    std::vector<ServiceConfig> services;  // 服务配置列表
    double update_rate = 50.0;            // 定时器更新频率（Hz）
};
```

**TopicConfig结构**：
```cpp
struct TopicConfig {
    std::string name;                    // 话题名称标识
    std::vector<std::string> write_registers;  // 写入寄存器列表
    std::vector<std::string> read_registers;  // 读取寄存器列表
    std::string command_topic;           // 命令话题名称
    std::string state_topic;             // 状态话题名称
    int touch_version = 1;               // 触觉数据版本号
};
```

**ServiceConfig结构**：
```cpp
struct ServiceConfig {
    std::string register_name;           // 寄存器名称
    std::string set_service_name;        // Set服务名称
    std::string get_service_name;        // Get服务名称
    bool is_write_register;              // 是否为写入寄存器
};
```

### 8.4 使用示例

**配置文件 (ros2_controller_config.yaml)**：

```yaml
device_nodes:
  - device: hand_left
    # interfaces_profile 由 device_protocol_config.yaml 的 protocol.type 自动推导，勿在此填写
    update_rate: 50
    publish_header:
      frame_id: "hand_left"
    joint_names:
      - "hand_left/joint_0"
      # ... RH5DG2 共 13 项；RH56F1 共 6 项

    topics:
      - name: angle_control
        registers:
          write: ["angleSet"]
          read: ["angleAct"]
        command_topic: "/hand_left/angle_set"
        state_topic: "/hand_left/angle_actual"

    services:
      - register_name: "angleSet"
        set_service_name: "/hand_left/set_angle"
        is_write_register: true
```

以下为 **`protocol.type`** 为 RH5DG2 系列时的命令示例（**RH56F1** 系列请改用 **`rh56f1_interfaces`**，且关节数为 **6**）。

**ROS2 话题使用**：

```bash
# 发布角度命令（13 维 joint_values）
ros2 topic pub --once /hand_left/angle_set rh5dg2_interfaces/msg/SetAngle1 \
  "{hand_id: 1, joint_values: [100,100,100,100,100,100,100,100,100,100,100,100,100]}"

ros2 topic echo /hand_left/angle_actual
```

**ROS2 服务使用**：

```bash
ros2 service call /hand_left/set_angle rh5dg2_interfaces/srv/Setangle \
  "{command: '', hand_id: 1, joint_values: [100,100,100,100,100,100,100,100,100,100,100,100,100]}"

ros2 service call /hand_left/get_errorCode rh5dg2_interfaces/srv/Geterror \
  "{query: '', hand_id: 1}"
```

### 8.5 数据流和通信方式

**话题模式数据流**：
```
定时器循环 (50Hz)
  → controlLoop()
  → readRegister() (读取寄存器)
  → publishRegisterData() (发布状态话题)
  
订阅者发布命令
  → Subscriber回调
  → handleCommand()
  → writeRegister() (写入寄存器)
```

**服务模式数据流**：
```
服务客户端调用
  → Service回调
  → pauseTimer(3ms) (暂停定时器)
  → readRegister() / writeRegister()
  → 返回服务响应
  → 恢复定时器
```

## 9. ROS2主节点 (inspire_control_node)

### 9.1 功能简介

`inspire_control_node`是ROS2主程序入口，负责初始化ROS2、加载配置、创建设备和控制器，并运行ROS2事件循环。

### 9.2 主要功能

1. **初始化ROS2**：`rclcpp::init()`
2. **加载配置文件**：设备配置和ROS2控制器配置
3. **配置日志系统**：从配置文件加载日志配置
4. **创建设备管理器**：管理所有串口设备
5. **创建协议对象**：为每个设备创建独立的协议实例
6. **创建控制器节点**：为每个设备创建RegisterController节点
7. **运行事件循环**：`MultiThreadedExecutor::spin_some()`

### 9.3 配置参数

**命令行参数**：
- `--device-config`：设备协议配置文件路径
- `--controller-config`：ROS2控制器配置文件路径
- `--device`：单设备模式，指定设备名称

### 9.4 使用示例

**编译 ROS2 工作区**：

```bash
cd /home/ubuntu/serial_control/src/ros2
source /opt/ros/humble/setup.bash
colcon build --packages-select rh5dg2_interfaces rh56f1_interfaces inspire_control_ros2
source install/setup.bash
```

**启动单设备节点**：

```bash
  --controller-config /path/to/ros2_controller_config.yaml \
  --device hand_left
```

**启动多设备节点**：

```bash

ros2 launch inspire_control_ros2 inspire_control_multi_device.launch.py

# 或直接运行节点（不指定--device参数）
```

### 9.5 数据流和通信方式

**启动流程**：
```
程序启动
  → 初始化ROS2
  → 加载配置文件
  → 配置日志系统
  → 创建设备管理器
  → 创建协议对象
  → 创建控制器节点
  → 启动定时器循环
  → 运行ROS2事件循环
```

**运行流程**：
```
ROS2事件循环
  ├─ 定时器回调 (50Hz)
  │  └─ controlLoop() (读取寄存器并发布)
  ├─ 话题订阅回调
  │  └─ handleCommand() (写入寄存器)
  └─ 服务回调
     └─ 各寄存器对应的专用 .srv（由适配器创建，类型见 rh*_interfaces）
```
